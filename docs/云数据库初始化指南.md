# 家计通 · 云数据库初始化操作指南

> 本指南手把手教你创建全部集合、索引、权限规则。
> 预计耗时 20-30 分钟。操作环境：微信开发者工具 → 云开发控制台。

---

## 一、前置条件

1. 已在微信开发者工具开通云开发环境（参考 `.env.development` 的 `VITE_CLOUD_ENV`）
2. 打开微信开发者工具 → 顶部点击「云开发」→ 进入控制台 → 左侧选「数据库」

---

## 二、创建 13 个集合

在 数据库 → 集合列表页，点「+ 添加集合」，逐个创建以下 13 个集合。

**集合名称必须完全一致（小写 + 下划线）**：

| 序号 | 集合名称 | 用途 |
|------|----------|------|
| 1 | `users` | 用户主表 |
| 2 | `families` | 家庭 |
| 3 | `family_members` | 成员关系 |
| 4 | `financial_profiles` | 财务档案 |
| 5 | `budget_plans` | 预算方案/规划书 |
| 6 | `weekly_entries` | 周度填报 |
| 7 | `subscriptions` | 订阅权益 |
| 8 | `orders` | 支付订单 |
| 9 | `recommendation_status` | 建议采纳状态 |
| 10 | `family_invites` | 伴侣邀请 |
| 11 | `calc_sessions` | 免登录测算暂存 |
| 12 | `analytics_events` | 埋点 |
| 13 | `app_config` | 运营配置 |

### 操作步骤（以 `users` 为例，其余重复）

1. 数据库页面点「+ 添加集合」
2. 集合名称填：`users`
3. 点「确定」
4. 重复 1-3，依次创建其余 12 个集合

---

## 三、设置权限（13 个集合全部相同）

**所有集合权限统一设为「仅创建者可读写」的更严格版本 —— 实际用「自定义安全规则」设为全 false**，因为所有读写都经云函数（管理员权限）执行，客户端禁止直连。

### 操作步骤（每个集合都要做）

1. 数据库 → 点击集合名（如 `users`）
2. 右上角点「权限设置」（或「数据权限」）
3. 选择「自定义安全规则」
4. 将内容改为：

```json
{
  "read": false,
  "write": false
}
```

5. 点「保存」

> **为什么要全 false？** 技术方案 §4.2：业务库禁止客户端直连，所有数据读写经云函数 `cloud.database()`（管理员权限）执行，防止越权访问他人家庭数据。

---

## 四、创建索引（17 条）

在 数据库 → 点击集合名 → 「索引管理」标签页 → 「+ 添加索引」。

### 索引清单总表

| 序号 | 集合 | 索引名（自定义） | 字段名 | 是否唯一 |
|------|------|------------------|--------|----------|
| 1 | `users` | `idx_openid` | `_openid` | ✅ 是 |
| 2 | `families` | `idx_owner_openid` | `owner_openid` | 否 |
| 3 | `family_members` | `idx_family_openid` | `family_id`, `openid` | ✅ 是 |
| 4 | `financial_profiles` | `idx_family_id` | `family_id` | ✅ 是 |
| 5 | `budget_plans` | `idx_family_active` | `family_id`, `is_active` | 否 |
| 6 | `budget_plans` | `idx_family_created` | `family_id`, `created_at` | 否 |
| 7 | `weekly_entries` | `idx_family_week` | `family_id`, `week_start` | ✅ 是 |
| 8 | `recommendation_status` | `idx_family_plan_rule` | `family_id`, `plan_id`, `rule_id` | ✅ 是 |
| 9 | `subscriptions` | `idx_openid` | `openid` | 否 |
| 10 | `subscriptions` | `idx_family_id` | `family_id` | 否 |
| 11 | `orders` | `idx_openid_created` | `openid`, `created_at` | 否 |
| 12 | `family_invites` | `idx_invite_code` | `invite_code` | ✅ 是 |
| 13 | `calc_sessions` | `idx_openid_created` | `_openid`, `created_at` | 否 |
| 14 | `calc_sessions` | `idx_expire_at` | `expire_at` | 否 |
| 15 | `analytics_events` | `idx_openid_created` | `_openid`, `created_at` | 否 |
| 16 | `analytics_events` | `idx_event_created` | `event`, `created_at` | 否 |
| 17 | `app_config` | `idx_key` | `key` | ✅ 是 |

### 操作步骤（以 `users` 集合 `_openid` 唯一索引为例）

1. 数据库 → 点击 `users` 集合
2. 点「索引管理」标签
3. 点「+ 添加索引」
4. 填写：
   - **索引名**：`idx_openid`（自定义名称，随便填，但要有意义）
   - **字段名**：点「+ 添加字段」，字段名填 `_openid`，排序选「升序」
   - **是否唯一**：✅ 勾选「是」
5. 点「确定」

### 复合索引操作步骤（以 `family_members` 的 `family_id + openid` 为例）

1. 数据库 → 点击 `family_members` 集合 → 索引管理 → 「+ 添加索引」
2. 填写：
   - **索引名**：`idx_family_openid`
   - **字段**：
     - 第 1 行：字段名 `family_id`，排序「升序」
     - 点「+ 添加字段」
     - 第 2 行：字段名 `openid`，排序「升序」
   - **是否唯一**：✅ 勾选「是」
3. 点「确定」

> 重复上述操作，按总表创建全部 17 条索引。

## 七、常见问题

**Q: 权限设为全 false，云函数还能读写吗？**
A: 可以。云函数使用 `cloud.database()` 是管理员权限，不受安全规则限制。安全规则只约束客户端 `wx.cloud.database()` 直连。

**Q: 唯一索引创建失败怎么办？**
A: 通常是因为集合里已有重复数据。先清空集合数据（删除全部记录），再创建唯一索引。

**Q: 索引名必须和文档一致吗？**
A: 不必须。索引名只是标识，关键是「字段名」和「是否唯一」要正确。

**Q: 创建完集合和索引后还需要做什么？**
A: 部署云函数（右键 `cloudfunctions/api/` → 「上传并部署：云端安装依赖」），然后在开发者工具控制台验证：
```javascript
wx.cloud.callFunction({ name: 'api', data: { action: 'cities.list' } })
  .then(res => console.log(res.result))
```
应返回 10 个城市。

## 一、产品愿景与定位

### 1.1 产品名称

**家计通**（JiaJiTong）—— 楔子版 Slogan：

> **「刚结婚，准备要娃？3 分钟知道你家合理预算是多少。」**

### 1.2 一句话定位

面向 **25–32 岁、一线/新一线城市、新婚或备育中的双职工家庭**，在人生财务结构剧变期（合并收支、房贷、筹备育儿），提供 **「科学预算测算 → 可执行规划报告 → 轻量执行追踪」** 的家庭财务教练服务。

### 1.3 我们要解决的核心问题

用户在这一人生阶段的真实困惑：

| 用户原话 | 产品回应 |
|----------|----------|
| 「刚结婚，不知道两个人合起来一个月花多少才合理」 | 基于城市 + 收入 + 家庭结构的弹性预算区间 |
| 「准备要娃，不知道还要攒多少钱才够」 | 备育专项储备测算 + 首年育儿预算增量 |
| 「房贷占了一半，还能不能生？」 | 固定支出占比分析 + 抗风险评级 |
| 「不想天天记账，但想知道有没有花超」 | 周度 7 大类填报 + 预算进度看板 |
| 「想让另一半也看看，一起商量」 | 伴侣邀请 + 共享规划书 |

### 1.4 我们不做什么（楔子版边界）

- ❌ 不做全能记账 App（不与随手记、咔皮记账正面竞争）
- ❌ 不做 AI 对话记账（叙事差异：我们卖「规划结果」，不卖「AI 记流水」）
- ❌ 不做理财交易、信贷导流（MVP 及前 6 个月）
- ❌ 不服务单身职场新人、自由职业等泛人群（二期再扩展）

### 1.5 产品方法论：「规划先行，追踪跟上」

```
传统记账 App：  记账 → 看花了多少 → 设预算（事后）
家计通楔子版：  输入家庭信息 → 系统给出合理预算 → 轻量追踪执行（事前）
```

这与海外 YNAB「方法论先行」逻辑一致，但落地场景聚焦 **中国家庭备育节点**，并用 **城市消费基准** 降低用户认知门槛。

### 1.6 MVP 成功标准

| 指标 | 目标 | 说明 |
|------|------|------|
| 首次测算完成率 | ≥ 75% | 进入测算流程后完成报告 |
| 报告分享率 | ≥ 15% | 生成规划书长图/链接分享 |
| 伴侣邀请率 | ≥ 20% | 完成测算后邀请配偶查看 |
| 7 日留存 | ≥ 35% | 测算后 7 天内再次打开 |
| **3 个月付费验证** | **≥ 100 人付 ¥9.9–19.9** | 商业模式成立信号 |

## 二、目标用户与场景

### 2.1 楔子用户定义（ICP）

**Ideal Customer Profile**：

| 属性 | 描述 |
|------|------|
| 年龄 | 25–32 岁 |
| 婚姻状态 | 新婚（0–2 年）或备育中（计划 1–2 年内生育） |
| 城市 | 一线 / 新一线（首期：上海、北京、深圳、杭州、成都、武汉、南京、苏州、西安、重庆） |
| 收入 | 家庭税后月收入 ¥15,000 – ¥50,000 |
| 住房 | 有房贷或月租 ≥ ¥3,000 |
| 家庭结构 | 二人世界，或已婚无孩准备要娃 |
| 行为特征 | 有储蓄意愿、关注小红书/知乎备育内容、伴侣共同决策 |

**明确排除（MVP 不做）**：单身、已育（孩子 > 3 岁）、三线及以下城市、家庭月收入 < ¥10,000 或 > ¥80,000。

### 2.3 用户任务（Jobs To Be Done）

| 任务 | 情境 | 情感 |
|------|------|------|
| **Functional** | 算出家庭合理月预算 | 从不确定 → 有依据 |
| **Functional** | 知道备育还需攒多少钱 | 从焦虑 → 有计划 |
| **Social** | 与配偶对齐财务预期 | 从各花各的 → 共同目标 |
| **Emotional** | 确认「我们能生得起/过得好」 | 从恐慌 → 安心 |

## 四、核心用户流程

### 4.1 主流程：首次规划（Happy Path）

```
落地页(P-01)
  → 点击「3 分钟测算」
  → 免登录快测(P-02) [可选，降低门槛]
      或
  → 微信授权登录
  → 测算向导(P-03~P-07)，约 3–5 分钟
  → 规划书预览(P-08)
      → 免费看到：健康分、总预算、1 条建议
      → 付费解锁：完整预算表、全部建议、PDF
  → 确认「启用此预算」
  → 引导：邀请伴侣(P-14) / 设置周度提醒 / 分享长图(P-10)
  → 进入看板(P-11)
```

**时长目标**：从进入到看到免费报告摘要 ≤ **3 分钟**。

### 4.2 流程：免登录快测（引流转化）

为内容渠道（小红书、知乎）设计 **零门槛体验**：

| 步骤 | 输入 | 输出 |
|------|------|------|
| 1 | 城市（下拉） | — |
| 2 | 家庭月收入（滑块/输入） | — |
| 3 | 房贷或房租（输入） | 简化报告：合理生活预算区间、健康分、1 条建议 |
| 4 | 引导登录 | 「查看完整规划书，保存并追踪执行」 |

免登录快测 **不保存数据**；登录后可一键带入已填信息进入完整向导。

### 4.3 流程：周度追踪

```
每周日 20:00 微信订阅消息
  → 打开小程序 → 周度填报(P-12)
  → 填写 7 大类本周支出（约 30 秒）
  → 自动更新看板(P-11)
  → 若某类超支 → 推送行动建议
```

### 4.4 流程：备育储备追踪（备育用户专属）

```
完成测算时设定：计划生育日期 / 目标储备金额
  → 首页展示「备育储备进度条」
  → 每月复盘：距离目标还差多少、按当前储蓄率是否达标
  → 若储蓄率不足 → 触发 R-N03 建议
```

### 4.5 流程：伴侣协同

```
用户 A 完成规划 → 点击「邀请伴侣」
  → 生成微信分享卡片（含家庭名称脱敏）
  → 伴侣 B 点击 → 微信登录 → 自动关联同一家庭
  → B 获得只读权限：看板 + 规划书
  → A 可开启「共同确认预算」（双方点击确认）
```

## 五、页面详细规格

### 5.1 P-01 落地页

**目标**：3 秒内传达价值，驱动点击测算。

| 元素 | 内容 |
|------|------|
| 主标题 | 刚结婚 / 准备要娃？3 分钟知道合理家庭预算 |
| 副标题 | 基于你的城市、收入、家庭结构，科学测算 |
| 社会证明 | 「已为 XXX 个家庭生成规划书」（内测后启用） |
| 主 CTA | 「免费开始测算」 |
| 次 CTA | 「已有规划书？登录查看」 |
| 信任背书 | 数据本地加密 · 不算理财 · 不卖贷款 |

**视觉**：温暖家庭感（非金融冷色调）；可用插画：年轻夫妻 + 简约家居。

### 5.2 P-03 ~ P-07 测算向导

#### Step 1：家庭阶段（P-03）

| 字段 | 类型 | 选项 |
|------|------|------|
| 当前阶段 | 单选卡片 | ① 新婚磨合期（无生育计划） ② 备孕中 ③ 计划 N 年内生育（选年份） |
| 家庭人数 | 自动 | 固定 2 人（MVP） |

**备育分支**：选 ② 或 ③ 时，后续 Step 4 展示备育储备模块。

#### Step 2：城市与收入（P-04）

| 字段 | 类型 | 说明 |
|------|------|------|
| 所在城市 | 级联选择 | 首期 10 城，其他「即将开放」 |
| 家庭月收入 | 数字输入 | 税后；分「我的收入」「配偶收入」或合并 |
| 收入稳定性 | 单选 | 稳定 / 含奖金 / 一方波动（影响保守系数 0.9–1.0） |

#### Step 3：固定开销（P-05）

| 字段 | 类型 | 必填 |
|------|------|------|
| 房贷或房租 | 金额 | ✅ |
| 车贷 / 停车 | 金额 | — |
| 保险（含社保补充） | 金额 | — |
| 赡养父母 | 金额 | — |
| 其他固定项 | 可添加行 | — |

**交互**：实时显示「固定支出占收入比」；> 50% 时黄色警告。

#### Step 4：储蓄与备育目标（P-06）

| 字段 | 类型 | 说明 |
|------|------|------|
| 每月储蓄目标 | 金额或 % | 默认建议 20% |
| 当前应急金 | 金额 | 用于计算备用金月数 |
| **备育储备目标**（备育用户） | 金额 | 系统默认推荐：同城备育首年增量 × 6 个月缓冲 |
| **计划生育时间**（备育用户） | 年月 | 用于倒计时与进度 |

**备育储备默认公式（MVP）**：

```
推荐储备金 = 同城育儿首年增量成本 × 6 个月 + 产检分娩预留（城市系数）
育儿首年增量 = 奶粉尿布 + 早教托育 + 医疗 + 育儿用品（查基准库）
```

#### Step 5：确认生成（P-07）

- 汇总卡片：收入 / 固定 / 储蓄 / 可支配
- 按钮：「生成我的家庭规划书」
- 加载态：3 步动画（匹配城市数据 → 推算预算 → 生成建议），≤ 2 秒

### 5.3 P-08 / P-09 规划书

**规划书结构**（《家庭财务规划书》）：

```
1. 封面
   - 家庭财务健康分（0–100）
   - 抗风险评级（🟢🟡🔴）
   - 生成日期、城市、家庭阶段

2. 收支总览
   - 月收入 / 固定支出 / 储蓄 / 可支配
   - 固定支出占比、储蓄率

3. 月度预算建议表
   - 7 大类：餐饮、日用、娱乐、医疗、服饰、交通通讯、其他
   - 每类：建议金额 | 合理区间 | 占收入比
   - 可展开「计算依据」（基准 × 系数）

4. 备育专项（备育用户）
   - 首年育儿预算增量
   - 储备金目标 vs 当前进度
   - 距计划生育时间倒计时

5. 优化建议（Top 3–5）
   - 问题 → 行动 → 预估影响

6. 抗风险报告
   - 备用金月数、固定支出占比、储蓄率
   - 三色评级说明

7. 下一步行动
   - □ 启用预算追踪
   - □ 邀请伴侣
   - □ 设置周度提醒
```

### 6.2 计算逻辑

#### 6.2.1 基础公式（继承 PRD，楔子版微调）

```
可支配收入 = 家庭月总收入 − 固定开销合计 − 储蓄目标

类目基准 = 城市人均类目消费 × 家庭消费系数 × 城市等级系数 × 收入系数

类目建议值 = 类目基准（归一化使合计 = 可支配收入 × 0.95）
类目区间 = [建议值 × 0.75, 建议值 × 1.25]
```

#### 6.2.2 楔子版专属系数

**家庭阶段系数**（MVP 新增）：

| 阶段 | 系数 | 说明 |
|------|------|------|
| 新婚磨合期 | 1.0 | 基准 |
| 备孕中 | 1.05 | 医疗、营养增量 |
| 计划 1 年内生育 | 1.08 | 提前储备导向 |

**家庭消费系数**（二人世界固定）：

| 结构 | 系数 |
|------|------|
| 二人世界 | 1.8 |

**收入系数**（同城差异化）：

```
收入系数 = clamp(家庭月收入 / 同城该阶段中位数收入, 0.85, 1.15)
```

#### 6.2.3 健康分计算（0–100）

| 维度 | 权重 | 评分逻辑 |
|------|------|----------|
| 储蓄率 | 30% | ≥ 20% → 满分；10–20% → 线性；< 10% → 低分 |
| 固定支出占比 | 25% | ≤ 40% → 满分；40–50% → 中；> 50% → 低 |
| 备用金月数 | 25% | ≥ 6 月 → 满分；3–6 → 中；< 3 → 低 |
| 预算结构合理性 | 20% | 餐饮、娱乐是否在区间内 |

**抗风险评级**：
- 🟢 健康分 ≥ 75 且备用金 ≥ 3 月
- 🟡 健康分 50–74 或固定支出 45–55%
- 🔴 健康分 < 50 或固定支出 > 55% 或备用金 < 1 月

#### 6.2.4 备育储备测算

```
首年育儿增量（月）= 奶粉尿布 + 早教托育 + 育儿医疗 + 育儿用品（查备育基准库）
推荐储备金 = 首年育儿增量 × 12 + 产检分娩一次性（城市系数）+ 6 个月生活缓冲
每月应攒 = (推荐储备金 − 当前已攒) / 距生育月数
```

若 `每月应攒 + 储蓄目标 > 可支配收入 × 30%`，触发 **R-N01 备育储备压力** 建议。

### 6.3 异常处理

| 场景 | 处理 |
|------|------|
| 固定支出 ≥ 收入 90% | 阻断，展示「收支失衡」页 + 3 条削减方向 |
| 城市未覆盖 | 「您的城市即将开放，暂按新一线标准估算」+ 标注 |
| 收入超出预设范围 | 允许继续，标注「个性化程度可能降低」 |
| 备育时间 < 6 个月且储备不足 | 红色预警 + 优先建议 |

### 7.1 建议规则：通用（继承 PRD R01–R07）

### 7.2 建议规则：备育/新婚专属（R-N 系列）

| 规则 ID | 触发条件 | 建议示例 |
|---------|----------|----------|
| R-N01 备育储备压力 | 每月应攒 + 原储蓄目标 > 可支配 30% | 「按计划在 12 个月后生育，每月需额外储备 ¥4,000，建议：将娱乐预算压缩 20%，或调整生育时间线」 |

| R-N06 医疗预算偏低 | 医疗预算 < 基准下限 且阶段 = 备育 | 「备孕阶段建议提高医疗预算，含产检预留 ¥X」 |

# Phase 6：微信登录 + 报告持久化 实现计划

**Goal:** 用户能微信一键登录，登录后 plan 自动入库并跨会话可见。M5 里程碑达成。

**Architecture:**
- 云函数 `api` 网关扩 3 个 action：`user.bootstrap`、`plan.save`、`plan.getActive`
- 云数据库 4 个集合：`users`、`families`、`financial_profiles`、`budget_plans`
- 前端 `pages/login/index.vue` 走 `wx.login` → `auth.wxLogin`（user.bootstrap）→ 缓存 user/family_id
- wizard step5 「生成」按钮后: 调 `plan.save` 把 calc.full 结果入库
- preview 页从云端 `plan.getActive` 拉数据（不再依赖 globalData）

**Tech Stack:** CommonJS 云函数 + `wx-server-sdk` + Node 单测。

## Global Constraints

- 4 个集合 + 索引一次创建（在云函数 cold start 时建或 `scripts/create-collections.js` 手工跑）
- 敏感字段（income）暂不加密（MVP 简化；Phase 10 接 AES-256）
- openid 在云函数 ctx 拿（wx-server-sdk 自动注入 `WXContext.OPENID`），前端不传
- 月度填写 / 伴侣邀请 / 支付 都不在本 phase

### Task 1: create-collections.js

**Files:** `scripts/create-collections.js`

**行为：** 通过 wx-server-sdk 在云开发环境创建 4 个集合 + 索引

- users: `_openid` 唯一索引
- families: `_id` 默认
- financial_profiles: `family_id` 唯一索引
- budget_plans: `(family_id, is_active)` 联合索引 + `(family_id, created_at)` 降序

**注意：** 本脚本需要真实 envId + appid，**M5 验证时可手工跑**。
